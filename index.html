Capstone Project 5: Event Registration & Ticketing System

A production-grade REST API for event registration and ticketing (similar to Eventbrite), designed to handle high concurrency and prevent overbooking using database-level locking and transactional guarantees.

Project Deliverables Implemented

Complete REST API

  • Event creation and management endpoints
  • User registration for events with limited capacity
  • Event browsing and registration tracking
  • Health check endpoint for service monitoring

Database Schema & Constraints

  • PostgreSQL schema with proper foreign key relationships
  • Capacity constraints enforced at the database level
  • ACID-compliant transactions
  • Row-level locking for safe seat allocation

Concurrent Booking Simulation

  • Automated test simulating 20 concurrent users
  • Single available slot scenario
  • Expected outcome: 1 success, remaining controlled failures
  • Validates race-condition resistance

Race Condition Prevention Strategy

  • Uses SELECT FOR UPDATE for pessimistic locking
  • Wraps booking flow inside database transactions
  • Atomic seat decrement logic
  • Rollback mechanism on failure
  • Ensures zero overbooking under concurrent load

Documentation

  • Comprehensive README with API reference
  • Technical deep dive into concurrency strategy
  • Clear explanation of locking model and transaction flow
